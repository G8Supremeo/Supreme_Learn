What is a Custom Hook?

A custom hook is simply a regular JavaScript function whose name starts with use. It allows you to extract, share, and reuse complex stateful logic across many components — without duplicating code or building class hierarchies.


SupreMify ships 3 custom hooks that power the entire application:

useLocalStorage useNotification useAuth (AuthContext)

Native Hooks Powering Our Custom Hooks

Before the custom hooks, these native React primitives are composed inside them:

  • useState — Stores mutable reactive state.
  • useEffect — Runs side-effects (subscriptions, timers).
  • useCallback — Memoises functions to prevent unnecessary re-creation on each render.
  • useContext — Reads from a shared React Context tree.
  • createContext — Initialises the global state container.

Custom Hook #1

useLocalStorage

Persists React state directly into the browser's localStorage. Data survives page refreshes and tab closures automatically — without manually calling localStorage.getItem everywhere.

useLocalStorage — Source Code


import { useState } from 'react';

export function useLocalStorage(key, initialValue) {
  // Reads existing localStorage value on first render
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      // Parse stored JSON or fall back to the initial value
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }
  });

  const setValue = (value) => {
    try {
      // Supports both plain values AND updater functions
      const valueToStore = value instanceof Function
        ? value(storedValue) : value;
      setStoredValue(valueToStore);
      // Sync React state AND actual localStorage simultaneously
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  };

  return [storedValue, setValue]; // Same API as useState
}

useLocalStorage — How It's Used

Any component drops in useLocalStorage exactly like useState — but data automatically survives a full page reload.


// Storing the user's active Dark Mode preference
const [theme, setTheme] = useLocalStorage('theme', 'light');

// Toggle it — React state AND localStorage both update
return (
  <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
    Switch to {theme === 'light' ? 'Dark' : 'Light'} Mode
  </button>
);

Primitive used inside: useState with a lazy initialiser function.

Custom Hook #2

useNotification

Manages the full lifecycle of temporary toast notification banners — triggering them, scheduling their auto-dismissal via a timer, and clearing them early if needed — all in one reusable unit.

useNotification — Source Code


import { useState, useCallback } from 'react';

export function useNotification() {
  const [notification, setNotification] = useState(null);

  // useCallback memoises this function — it is NOT recreated
  // on every single render, which prevents infinite effect loops
  const showNotification = useCallback((message, type = 'success', duration = 3000) => {
    setNotification({ message, type });
    
    // Automatically fires a timer to erase the notification
    setTimeout(() => {
      setNotification(null);
    }, duration);
  }, []); // Empty dep array — function identity is STABLE forever

  const hideNotification = useCallback(() => {
    setNotification(null);
  }, []);

  return { notification, showNotification, hideNotification };
}

useNotification — How It's Used

Used inside Profile.jsx to emit a branded success flash when the user saves their profile changes to Supabase.


import { useNotification } from '../hooks/useNotification';

export function Profile() {
  const { notification, showNotification } = useNotification();
  
  const handleSave = async () => {
    await updateProfile({ full_name, phone, avatar_url });
    
    // Fire a 3-second auto-dismissing success banner
    showNotification('Profile Updated Successfully!', 'success', 3000);
  };

  return (
    <>
      {notification && (
        <div className={`alert alert-${notification.type}`}>
          {notification.message}
        </div>
      )}
      <button onClick={handleSave}>Save Changes</button>
    </>
  );
}

Primitives used inside: useState + useCallback for stable function identity.

Custom Hook #3

useAuth (AuthContext)

The most powerful custom hook. Wraps a full React Context provider around the entire application to broadcast the Supabase live user session, Google OAuth methods, and profile updates down without prop-drilling.

useAuth — Architecture


// AuthContext.jsx — The Global Session Engine
const AuthContext = createContext();

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Subscribe to Supabase: fires on login, logout, token refresh
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (event, session) => {
        setUser(session?.user ?? null);
        setLoading(false);
      }
    );
    return () => subscription.unsubscribe(); // Cleanup on unmount!
  }, []);

  // Google OAuth launcher
  const signInWithGoogle = async () => {
    await supabase.auth.signInWithOAuth({ provider: 'google' });
  };

  return (
    <AuthContext.Provider value={{ user, loading, signInWithGoogle, signOut }}>
      {children}
    </AuthContext.Provider>
  );
}

// The Custom Hook wrapper — clean API for any component to consume
export function useAuth() {
  return useContext(AuthContext);
}

useAuth — How It's Used Everywhere

Any component anywhere in the tree calls useAuth() to instantly receive the live session — no props needed.


// Navbar.jsx — Rendering the user avatar and logout button
const { user, signOut } = useAuth();

// Landing.jsx — Auto-redirecting logged-in users to the dashboard
const { isAuthenticated, loading } = useAuth();
if (isAuthenticated) return <Navigate to="/dashboard" replace />;

// PrivateRoute.jsx — Protecting every authenticated page
const { user, loading } = useAuth();
if (!user) return <Navigate to="/login" replace />;

// Profile.jsx — Reading and saving Supabase user metadata
const { user, updateProfile } = useAuth();

Primitives used inside: createContext, useContext, useState, useEffect.

Hook Composition Map

How React's native hooks are composed inside our custom ones:

Conclusion

SupreMify demonstrates that powerful, production-grade platforms can be built with a small, deliberately composed set of custom React hooks rather than heavy state management libraries like Redux.


Three custom hooks orchestrate persistent storage, real-time notifications, and a live global authentication pipeline — all from pure, composable React functions.
